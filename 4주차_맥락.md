## RHT-01 센서 관련 메모

- MaxDetect 1-Wire 프로토콜 사용 (Dallas 1-Wire와 호환 안 됨)
- 전원: 3.3V~6V / 측정 간격: 2초 이상
- Operating range: 20~90%RH / 0~50℃
- Accuracy: ±5%RH / ±2℃ — 절대 정밀도보다 상대 변화 감지 용도로 사용
- 15년 이상 된 센서라 노화(±1%RH/year) 반영하여 5회 중간값 필터 적용
- 온습도는 임계값 제어에 사용하지 않으므로 오차는 허용 범위

## 주요 구현 내용

### delay_us (DWT 기반)

```c
static void delay_us(uint32_t us) {
    uint32_t start = DWT->CYCCNT;
    uint32_t ticks = us * (SystemCoreClock / 1000000U);
    while ((DWT->CYCCNT - start) < ticks);
}
```

- HAL_Delay()는 ms 단위라 수십 us 타이밍 제어 불가 → DWT CYCCNT 사용
- 뺄셈 방식으로 32비트 overflow에도 올바르게 동작
- DWT 활성화는 main.c USER CODE BEGIN 2에서 수행

### DWT 활성화 (main.c USER CODE BEGIN 2)

```c
CoreDebug->DEMCR |= CoreDebug_DEMCR_TRCENA_Msk;
DWT->CYCCNT = 0;
DWT->CTRL   |= DWT_CTRL_CYCCNTENA_Msk;
```

### 통신 프로토콜 요약

```
MCU → 센서: 5ms LOW → 30us HIGH (Start Signal)
센서 → MCU: 80us LOW → 80us HIGH (응답)
데이터:     50us LOW → 26~28us HIGH = "0" / 70us HIGH = "1"
총 40비트:  16bit 습도 + 16bit 온도 + 8bit 체크섬
```

### 데이터 변환

```c
data->humidity    = bytes[0] + bytes[1] * 0.1f;
// 온도 MSB = 1이면 영하
if (bytes[2] & 0x80)
    data->temperature = -((bytes[2] & 0x7F) + bytes[3] * 0.1f);
else
    data->temperature = bytes[2] + bytes[3] * 0.1f;
```

### 5회 중간값 필터

```c
HAL_StatusTypeDef RHT01_ReadMedian(RHT01_Data *data);
```

- 5회 읽기 (2초 간격 = 총 약 10초)
- hum 기준 버블 정렬 후 중간값 선택 (temp는 hum과 쌍으로 이동)
- 1회 이상 성공하면 OK, 전부 실패 시 HAL_ERROR 반환

## 이번 주 배운 것들

- 비트뱅잉: GPIO를 OUTPUT ↔ INPUT으로 전환하며 직접 프로토콜 구현
- DWT(Data Watchpoint and Trace): CPU 클럭 카운터로 us 단위 딜레이 구현
- 32비트 unsigned 뺄셈의 overflow 안전성 (wrap-around)
- Include Guard (`#ifndef / #define / #endif`) — 헤더 중복 포함 방지
- HAL_StatusTypeDef — HAL 함수 반환값 규약
- 센서 스펙 용어: Operating Range / Resolution / Accuracy / Repeatability / Long-term Stability
- CubeMX 자동 생성 영역은 코드에서 직접 수정 시 재생성으로 덮어씌워짐
