주차별 맥락 파일(`주차별_맥락/{N}주차_맥락.md`)을 업데이트합니다.

## 주차 번호 결정

- `$ARGUMENTS`가 있으면 그 숫자를 주차로 사용 (예: `/update-week 5` → 5주차)
- 없으면 `개발_일정_체크리스트.md`를 읽어서 현재 진행 중인 주차를 자동으로 파악

## 절차

1. **주차 번호 확정** 후 `주차별_맥락/{N}주차_맥락.md` 파일 확인
   - 파일이 있으면 읽어서 내용 파악
   - 없으면 직전 주차 파일을 형식 참고용으로 읽고 새로 생성

2. **git 정보 수집**
   - `git log --oneline -20` — 최근 커밋 목록
   - `git diff HEAD~3..HEAD --stat` — 최근 변경된 파일 목록
   - 이번 주차와 관련된 커밋만 필터링

3. **현재 소스 구조 파악**
   - BSP, App, Core 폴더 구조 확인
   - 새로 추가된 파일 확인

4. **맥락 파일 업데이트**
   - 현재 진행 상황 섹션 갱신 (완료/미완료 항목)
   - CubeMX 설정 추가 사항 (이번 세션에서 새로 설정한 것)
   - 오늘 대화에서 다룬 내용 반영:
     - 새로 배운 개념 및 설명
     - 구현한 기능
     - 발견한 문제 및 해결 방법
     - 실물 연결 및 테스트 결과
   - 파일 구조 섹션 업데이트
   - 이번 주 배운 것들 섹션 보완

5. **형식 규칙**
   - 이전 주차 파일 형식을 기준으로 유지
   - 아직 완료되지 않은 항목은 빈칸으로 두거나 "예정" 표기
   - 코드 블록, 타이밍 다이어그램 등 학습에 도움되는 내용 포함
   - 이번 주 배운 것들 최소한의 불필요한 내용만 거르고 너무 요약하지 말 것.
      - 나중에 문서만 보고 다시 공부할 수 있게. 또 주차별 맥락을 모아서 책으로도 쓸 수 있게 정리할 것.
