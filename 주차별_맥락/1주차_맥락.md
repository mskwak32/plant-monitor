## 이번 주 배운 것들

### STM32CubeIDE / STM32CubeMX 관계

STM32CubeMX는 핀 배치, 클럭, 주변장치 설정을 GUI로 구성하고 초기화 코드를 자동 생성하는 도구다. STM32CubeIDE는 CubeMX가 통합된 Eclipse 기반 IDE로, 코드 편집·빌드·디버깅을 모두 처리한다. `.ioc` 파일이 CubeMX 설정의 원본이며, 이 파일을 수정하고 "Generate Code"를 누르면 `main.c`, `stm32f1xx_hal_msp.c` 등이 재생성된다.

### HAL (Hardware Abstraction Layer)

STM32 HAL은 ST가 제공하는 공식 드라이버 라이브러리다. 레지스터를 직접 조작하는 대신 `HAL_GPIO_TogglePin()` 같은 함수를 호출하면 된다. MCU 시리즈가 달라도 HAL API가 거의 동일하게 유지되므로 코드 이식성이 높다. 단, 추상화 오버헤드가 있어 타이밍에 극도로 민감한 코드에는 레지스터 직접 접근이 필요할 수 있다.

### GPIO (General Purpose Input/Output)

MCU의 범용 입출력 핀. `Output Push-Pull` 모드로 설정하면 핀을 HIGH(3.3V) 또는 LOW(GND)로 능동적으로 구동할 수 있다. `HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5)`는 PA5 핀의 현재 상태를 반전시킨다.

### HAL_Delay

SysTick 타이머를 기반으로 ms 단위 블로킹 지연을 제공하는 함수. `HAL_Init()`가 SysTick을 1ms 주기로 설정하므로 `HAL_Delay(500)`은 500ms 대기한다. CPU가 그 시간 동안 아무 일도 하지 않고 기다리는 "블로킹" 방식이라, 나중에 다른 작업과 병행해야 할 때는 타이머 인터럽트 방식으로 교체해야 한다.

### 빌드·플래시 과정

STM32CubeIDE에서 "Build"를 누르면 C 소스가 ARM GCC 크로스 컴파일러로 컴파일·링크되어 `.elf` 파일이 생성된다. "Run(F11)"을 누르면 ST-Link가 USB를 통해 `.elf`를 Flash 메모리에 기록(플래시)하고 MCU를 리셋하여 코드가 실행된다. PC에서 실행 파일을 바로 실행하는 것과 달리, 대상 MCU 아키텍처(ARM Cortex-M3)용으로 크로스 컴파일해야 하며 플래시 단계가 반드시 필요하다.
