## 주요 구현 내용

### printf 리디렉션 (USER CODE BEGIN 0)

```c
#ifdef __GNUC__
#define PUTCHAR_PROTOTYPE int __io_putchar(int ch)
#else
#define PUTCHAR_PROTOTYPE int fputc(int ch, FILE *f)
#endif

PUTCHAR_PROTOTYPE {
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

### 메인 루프 (USER CODE BEGIN WHILE)

```c
printf("Hello, World!\r\n");
HAL_Delay(1000);
```

## 이번 주 배운 것들

### UART (Universal Asynchronous Receiver/Transmitter)

UART는 두 장치 사이에서 직렬(serial)로 데이터를 주고받는 통신 방식이다. "비동기(Asynchronous)"란 공유 클럭 없이 송수신 양측이 미리 약속한 속도(보레이트)로 통신한다는 뜻이다. 1비트씩 순서대로 전송하며, 한 프레임은 시작 비트(1) + 데이터 비트(8) + 정지 비트(1)로 구성된다.

**보레이트(Baud rate)** 는 초당 전송하는 심볼(비트) 수다. 115200bps는 초당 115,200비트를 전송한다는 뜻이며, 10비트 프레임 기준으로 초당 약 11,520 바이트를 전송할 수 있다. 양쪽 보레이트가 다르면 수신 측이 비트 경계를 잘못 판단해 데이터가 깨진다.

**TX / RX 교차 연결**: STM32의 TX(PA2)는 상대방의 RX로, STM32의 RX(PA3)는 상대방의 TX로 연결해야 한다. Nucleo 보드에서 PA2/PA3은 ST-Link 칩의 UART 브리지에 직접 연결되어 있어 USB 케이블 하나로 PC와 통신할 수 있다.

### ST-Link USB-UART 브리지

Nucleo 보드의 ST-Link 칩은 디버거 역할 외에 USB-UART 브리지 기능도 내장하고 있다. MCU의 USART2(PA2/PA3)가 이 브리지에 연결되어 있으므로, PC에서 STM32CubeIDE의 시리얼 모니터나 PuTTY 같은 터미널 프로그램을 열어 포트를 선택하면 MCU가 출력하는 UART 데이터를 바로 볼 수 있다. 별도의 USB-UART 변환 모듈이 필요 없다.

### printf 리디렉션

표준 C의 `printf()`는 원래 PC의 표준 출력(stdout)으로 문자를 내보내는 함수지만, MCU에는 stdout이 없다. STM32 + GCC(newlib) 환경에서는 `printf()`가 내부적으로 `__io_putchar()`를 한 글자씩 호출하도록 설계되어 있다. 따라서 이 함수를 UART 전송으로 재정의(override)하면 `printf()` 출력 전체가 UART로 나오게 된다.

```c
int __io_putchar(int ch) {
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

`#ifdef __GNUC__`는 GCC 컴파일러를 사용할 때만 `__io_putchar`를 재정의하고, Keil 같은 다른 컴파일러 환경에서는 `fputc`를 재정의하도록 분기하는 전처리기 조건문이다.

### HAL_UART_Transmit

```c
HAL_StatusTypeDef HAL_UART_Transmit(UART_HandleTypeDef *huart,
                                     const uint8_t *pData,
                                     uint16_t Size,
                                     uint32_t Timeout);
```

- `huart`: UART 핸들 포인터 (CubeMX가 생성한 `huart2` 사용)
- `pData`: 전송할 데이터의 시작 주소
- `Size`: 전송할 바이트 수
- `Timeout`: 최대 대기 시간(ms). `HAL_MAX_DELAY`는 사실상 무한 대기

`printf` 리디렉션에서는 문자 하나씩(`Size=1`) 전송하므로, `printf("Hello\r\n")`을 호출하면 이 함수가 7번 호출된다.

### `\r\n` vs `\n`

UART 터미널에서 줄바꿈을 올바르게 표시하려면 `\r\n`(CR+LF)을 사용해야 한다. `\n`(LF)만 보내면 일부 터미널에서 줄이 바뀌지 않고 커서만 아래로 이동하거나, 화면이 이상하게 표시된다. `\r`은 커서를 줄의 맨 앞으로 이동시키고(Carriage Return), `\n`은 다음 줄로 이동시킨다(Line Feed). 이 두 문자가 함께 와야 "줄바꿈 후 새 줄 맨 앞"이 된다.

### 보레이트 계산 원리

USART2는 APB1 클럭(36MHz) 도메인에 속한다. 보레이트 레지스터(BRR)에 분주비를 설정하면 원하는 보레이트가 나온다.

```
보레이트 = APB1_CLK / (16 × USARTDIV)
115200 = 36,000,000 / (16 × 19.53)
→ USARTDIV ≈ 19.53 (BRR에 정수+소수 분리하여 기록)
```

CubeMX가 자동으로 계산하여 설정해주므로 직접 계산할 필요는 없지만, 클럭이 바뀌면 BRR 값도 달라져야 한다는 원리를 이해해두면 나중에 보레이트 오류 디버깅에 도움이 된다.
