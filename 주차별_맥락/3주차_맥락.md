```c
HAL_ADCEx_Calibration_Start(&hadc1);
```

### 메인 루프 (USER CODE BEGIN WHILE)

```c
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY);
uint32_t adc_value = HAL_ADC_GetValue(&hadc1);
HAL_ADC_Stop(&hadc1);

float voltage = adc_value * 3.3f / 4095.0f;
printf("Soil ADC: %lu Voltage: %.2fV\r\n", adc_value, voltage);

HAL_Delay(1000);
```

## 이번 주 배운 것들

### CubeMX USER CODE 블록 규칙

CubeMX는 `.ioc` 파일을 기반으로 `main.c`, `stm32f1xx_it.c` 등 여러 파일을 자동 생성한다. 핀 설정이나 클럭을 바꾸고 코드를 재생성하면 이 파일들이 덮어써진다. 이때 사용자가 작성한 코드가 사라지지 않도록 CubeMX는 파일 곳곳에 `/* USER CODE BEGIN xxx */` ~ `/* USER CODE END xxx */` 블록을 만들어둔다. **이 블록 안에 작성한 코드만 재생성 후에도 보존된다.** 블록 밖에 코드를 작성하면 다음 재생성 시 삭제되므로 반드시 블록 안에 써야 한다.

### ADC (Analog-to-Digital Converter)

현실 세계의 아날로그 신호(연속적인 전압)를 MCU가 처리할 수 있는 디지털 숫자로 변환하는 장치. 토양 수분 센서는 수분량에 따라 0~3.3V 사이의 전압을 출력하는데, MCU는 전압을 직접 읽을 수 없으므로 ADC가 이를 숫자로 변환한다.

**12-bit** 해상도란 변환 결과가 0~4095(2¹²−1) 사이의 정수로 나온다는 뜻이다. 0이 0V, 4095가 3.3V에 해당하므로 전압으로 환산하면 `voltage = adc_value × 3.3 / 4095`이다.

**Single-ended** 방식은 핀 하나의 전압을 GND 기준으로 측정하는 방식이다. 반대 개념인 Differential은 두 핀의 전압 차이를 측정하는 방식으로, 노이즈에 더 강하지만 핀을 2개 사용한다. 일반 센서 읽기에는 Single-ended로 충분하다.

**폴링 방식**은 `HAL_ADC_Start()`로 변환을 시작하고, `HAL_ADC_PollForConversion()`으로 완료를 기다린 뒤, `HAL_ADC_GetValue()`로 값을 읽는 순서로 동작한다. CPU가 변환이 끝날 때까지 대기하는 방식이라 단순하지만, 그동안 다른 일을 할 수 없다. 더 복잡한 방식으로는 변환 완료 시 인터럽트를 발생시키는 방법(DMA/인터럽트 방식)이 있다.

### 클럭 트리 (Clock Tree)

STM32는 여러 주변장치(UART, ADC, GPIO 등)가 각기 다른 클럭 도메인에서 동작하며, 모든 클럭의 원천은 하나의 소스에서 시작된다.

```
[클럭 소스]
  HSE (8MHz, Bypass)  HSI (8MHz RC)
        └──────────────┘
                │ PLL (×9)
                ▼
  ┌─────── SYSCLK (72MHz) ───────┐
  │               │               │
FCLK           HCLK            (기타)
(72MHz)        (72MHz)
                  │
       ┌──────────┴──────────┐
     APB1                  APB2
   (36MHz)               (72MHz)
  UART2,SPI,I2C      ADC1,GPIO,UART1
                          │
                    ADC Prescaler
                      (12MHz)

[저속 클럭 소스]
  LSE (32.768kHz)  LSI (40kHz RC)
        └──────────┘
              │
             RTC
```

ADC 최대 클럭이 14MHz이므로 72MHz를 /6하여 12MHz로 맞춘 것이다. 클럭이 너무 빠르면 ADC가 오동작하고, 너무 느리면 변환 시간이 길어진다.

Nucleo-F103RB 보드는 ST-Link 칩이 8MHz 클럭을 MCU에 공급하므로 외부 크리스탈 없이 HSE를 사용할 수 있다(HSE Bypass 모드).

#### RCC (Reset and Clock Control)

RCC는 STM32에서 클럭 공급과 리셋을 총괄 관리하는 내부 주변장치다. CubeMX에서 "RCC"를 설정하면 결국 이 RCC 레지스터들을 어떻게 구성할지 정하는 것이고, 생성된 `SystemClock_Config()` 함수가 HAL 코드로 초기화한다.

RCC의 주요 역할:
- **클럭 소스 선택 및 활성화**: HSE, HSI, LSE, LSI, PLL 중 무엇을 켤지
- **PLL 설정**: 배수(PLLMUL), 입력 분주(PREDIV) 등
- **SYSCLK 소스 전환**: 부팅 초기에는 HSI로 동작하다가 PLL이 락(lock)되면 소스를 PLL로 전환
- **버스 프리스케일러**: AHB, APB1, APB2 각 분주비 설정
- **주변장치 클럭 게이팅**: 사용하지 않는 주변장치의 클럭을 끊어 소비전력 절감
- **리셋 관리**: 전원 리셋·핀 리셋·소프트 리셋 등 리셋 원인 기록, 주변장치 강제 리셋

```c
// 주변장치 클럭 게이팅 예시 — 이걸 빠뜨리면 레지스터에 써도 아무 반응이 없다
__HAL_RCC_GPIOA_CLK_ENABLE();
__HAL_RCC_ADC1_CLK_ENABLE();
```

CubeMX가 `MX_GPIO_Init()`, `MX_ADC1_Init()` 등을 생성할 때 이 매크로들을 자동으로 포함시켜 주므로 평소에는 직접 호출할 일이 없다. 그러나 "GPIO를 설정했는데 동작하지 않는다"는 문제의 가장 흔한 원인이 이 클럭 활성화 누락이다.

#### 클럭 소스: HSE / HSI / LSE / LSI

STM32에는 4가지 클럭 소스가 있다. 앞의 H/L은 High/Low-speed, 뒤의 E/I는 External/Internal을 뜻한다.

| 이름 | 속도 | 출처 | 특징 |
|------|------|------|------|
| **HSE** | 4~16MHz (보통 8MHz) | 외부 크리스탈 또는 외부 클럭 입력 | 정확도 높음. 이 프로젝트에서는 ST-Link가 8MHz 공급 (Bypass 모드) |
| **HSI** | 8MHz | MCU 내부 RC 발진기 | 별도 부품 불필요, 온도·전압에 따라 오차 ±1~2% 발생 |
| **LSE** | 32.768kHz | 외부 32kHz 크리스탈 | RTC 전용. 주파수가 정확하고 전력 소모 매우 낮음 |
| **LSI** | ~40kHz | MCU 내부 RC 발진기 | 외부 크리스탈 없을 때 RTC·워치독 타이머용으로 사용. 오차 큼 (±30%) |

PLL은 HSE 또는 HSI/2를 소스로 받아 배수하여 최대 72MHz의 SYSCLK을 만든다. SYSCLK을 높이려면 반드시 PLL을 거쳐야 한다.

#### SYSCLK

시스템 클럭. CPU와 AHB 버스의 클럭 원천이다. 클럭 소스(HSE, HSI, PLL 출력 중 하나)를 선택하여 SYSCLK으로 설정한다. STM32F103의 최대값은 72MHz이다. HCLK, APB1, APB2 등 모든 하위 클럭은 SYSCLK을 분주하여 만들어진다.

#### HCLK (AHB Clock)

SYSCLK을 AHB Prescaler로 나눈 클럭. CPU 코어, 메모리(SRAM/Flash), DMA가 이 클럭으로 동작한다. 이 프로젝트에서는 분주 없이 SYSCLK과 동일하게 72MHz로 설정했다. AHB Prescaler를 /2로 설정하면 36MHz가 된다. `HAL_GetHCLKFreq()`로 현재 값을 읽을 수 있다.

#### AHB 버스 (Advanced High-performance Bus)

AHB는 ARM이 설계한 AMBA(Advanced Microcontroller Bus Architecture) 버스 표준의 일부로, MCU 내부에서 가장 빠른 데이터 통로다. CPU, SRAM, Flash, DMA가 이 버스에 연결되어 HCLK 속도(72MHz)로 데이터를 주고받는다.

```
CPU Core ──┐
DMA       ─┤──── AHB 버스 (72MHz) ──── SRAM / Flash
GPIO 레지스─┘
```

APB 버스에 연결된 주변장치들(UART, SPI, I2C 등)은 AHB 속도를 직접 받지 못하므로, AHB와 APB 사이에 **APB 브릿지**가 놓여 분주된 클럭을 공급한다. 이 브릿지를 거치기 때문에 APB 주변장치 레지스터 접근은 SRAM 접근보다 레이턴시가 있다.

버스 명칭을 헷갈리지 않는 법: **AHB는 "고속 내부 버스"**, **APB는 "저속 주변장치 버스"**라고 기억하면 된다. HCLK이 AHB의 클럭, APB1/APB2가 각 APB 브릿지의 클럭이다.

#### FCLK (Free-running Clock)

ARM Cortex-M 코어 내부의 자유 구동 클럭. HCLK과 동일한 주파수로 동작하지만, CPU가 저전력 슬립(WFI/WFE) 상태에 들어가도 멈추지 않고 계속 동작한다. 인터럽트나 이벤트를 감지하여 CPU를 깨우는 데 사용된다. 외부에서 직접 제어하거나 설정하는 값은 아니며, HCLK 설정을 따른다.

#### APB1 / APB2

HCLK을 각각의 APB Prescaler로 나눈 주변장치 버스 클럭이다.

| 버스 | 프리스케일러 | 주파수 | 연결된 주변장치 |
|------|------------|--------|----------------|
| **APB1** (저속) | /2 | 36MHz | UART2~5, SPI2~3, I2C1~2, TIM2~4, DAC |
| **APB2** (고속) | /1 | 72MHz | UART1, SPI1, ADC1~3, GPIO, TIM1, EXTI |

APB1의 최대 클럭은 36MHz, APB2는 72MHz로 제한된다. APB1에 연결된 타이머(TIM2~4)는 APB1 프리스케일러가 1이 아닌 경우 자동으로 ×2가 적용되어 실제 72MHz로 동작한다.

#### RTC (Real-Time Clock)

초·분·시·일 등 실시간 시각을 추적하는 카운터. MCU 본체와 별개의 저전력 도메인(VBAT)에서 동작하므로, MCU 전원이 꺼져도 코인 배터리로 시각 유지가 가능하다. 클럭 소스로 LSE(32.768kHz 크리스탈, 정확)나 LSI(내부 RC, 부정확)를 선택할 수 있다. 이 프로젝트에서는 RTC를 사용하지 않지만, STM32에서 클럭 소스를 이야기할 때 LSE/LSI의 주 수요처가 RTC다.

#### RCC_OSC_IN / RCC_OSC_OUT vs RCC_OSC32_IN / RCC_OSC32_OUT

보드의 물리적인 핀 이름이다. 숫자(`32` 유무)로 주파수 대역이 구분된다.

| 핀 이름 | MCU 핀 | 주파수 대역 | 연결 대상 | Nucleo-F103RB에서 |
|---------|--------|-----------|----------|-----------------|
| `RCC_OSC_IN` | PD0 | 고속 (4~16MHz) | HSE 크리스탈 또는 외부 클럭 입력 | ST-Link MCO가 8MHz 공급 → **HSE Bypass 모드**, IN 핀만 사용 |
| `RCC_OSC_OUT` | PD1 | 고속 | HSE 크리스탈 (반대편) | Bypass 모드이므로 **미사용**, GPIO로 전용 가능 |
| `RCC_OSC32_IN` | PC14 | 저속 (32.768kHz) | LSE 크리스탈 | 보드 버전 **C-02 이상**: 32.768kHz 크리스탈(X2) 실장, Crystal 모드로 사용 |
| `RCC_OSC32_OUT` | PC15 | 저속 | LSE 크리스탈 (반대편) | 보드 버전 **C-02 이상**: 크리스탈 반대편 연결, Crystal 모드로 사용 |

> **보드 버전 확인**: Nucleo 보드 뒷면 스티커에 `MB1136 C-0X` 형태로 표기됨. C-01은 LSE 미장착(PC14/PC15가 GPIO), C-02 이상은 32kHz 크리스탈 실장. CubeMX에서 LSE를 "Crystal/Ceramic Resonator"로 설정할 수 있다면 C-02 이상.
내가 사용하는 보드는 C-05이고 32kHz 크리스탈이 X2위치에 실장되어있다.

**IN / OUT 핀의 의미**

두 핀이 한 쌍인 이유는 MCU 내부에 **반전 앰프(inverting amplifier)** 가 있기 때문이다. 크리스탈은 이 앰프의 입출력 사이에 놓여 피드백 루프를 형성하고, 그 공진 주파수에서 안정적인 발진이 시작된다.

```
MCU 내부
┌─────────────────────┐
│  [반전 앰프]         │
│  IN ──▶ 앰프 ──▶ OUT │
│   │               │ │
└───┼───────────────┼─┘
    │   [크리스탈]   │
    └────┤ X ├──────┘
  RCC_OSC_IN      RCC_OSC_OUT
  (PC14/PD0)     (PC15/PD1)
```

- **`_IN`**: 발진 신호가 MCU 안으로 들어오는 핀. 크리스탈의 한쪽 다리를 연결한다.
- **`_OUT`**: MCU 내부 앰프의 출력이 나가는 핀. 크리스탈의 반대쪽 다리를 연결하여 피드백 루프를 완성한다.

**HSE Bypass 모드**: 외부에서 이미 만들어진 클럭 신호(ST-Link MCO)를 `RCC_OSC_IN`에 직접 주입한다. MCU 내부 앰프는 비활성화되고, `RCC_OSC_OUT`(PD1)은 아무 역할도 하지 않으므로 일반 GPIO로 전용할 수 있다.

**LSE Crystal 모드**: Nucleo C-02 이상에서 PC14(OSC32_IN)~PC15(OSC32_OUT) 사이에 32.768kHz 크리스탈(X2)이 연결된다. CubeMX에서 LSE를 "Crystal/Ceramic Resonator"로 선택하면 이 핀들이 자동으로 OSC32 기능으로 잠긴다.

### PLL (Phase-Locked Loop)

PLL은 낮은 주파수의 클럭 소스를 받아 정수 배수의 더 높은 주파수를 안정적으로 만들어내는 아날로그 회로다. 단순히 신호를 빠르게 반복하는 것이 아니라, 내부 VCO(전압 제어 발진기)가 출력 신호를 만들고 분주기로 나눈 뒤 입력 신호와 위상을 비교하면서 차이가 0이 될 때까지 VCO 주파수를 조정하는 피드백 루프로 동작한다.

```
입력 클럭(8MHz) → [위상 비교기] → [VCO] → 출력(72MHz)
                        ↑                       |
                   [분주기 /9] ←─────────────────┘
```

STM32F103에서 PLL은 HSE(또는 HSI/2)를 소스로 받아 2~16배 배수(PLLMUL)를 적용한다. 이 프로젝트에서는 HSE 8MHz × PLLMUL 9 = 72MHz로 설정한다. 72MHz는 STM32F103RBT6의 최대 동작 클럭이므로 성능을 최대한 끌어올린 설정이다.

PLL을 거치지 않고 HSE(8MHz)를 SYSCLK으로 바로 사용할 수도 있지만, 그러면 CPU가 9배 느리게 동작하고 ADC 클럭도 훨씬 낮아진다. 반대로 PLLMUL을 너무 높게 설정하면 MCU의 최대 클럭을 초과하여 오동작한다.

### `&` (주소 연산자)와 `*` (포인터)

C에서 변수는 메모리의 특정 주소에 저장된다. `&변수`는 그 변수의 메모리 주소를 반환한다. HAL 함수들이 `&hadc1`처럼 핸들을 포인터로 받는 이유는, 함수 내부에서 구조체의 내용을 직접 수정해야 하기 때문이다. 값으로 전달하면 복사본이 전달되어 원본이 바뀌지 않는다.

```c
HAL_ADC_Start(&hadc1);   // hadc1의 주소를 전달 → 함수 내부에서 hadc1 상태 변경 가능
```

`*`는 두 가지 용도로 쓰인다. 포인터 변수를 선언할 때(`int *p`)와, 포인터가 가리키는 메모리의 값을 읽거나 쓸 때(`*p = 10`) 사용한다.

### printf 리디렉션

표준 C의 `printf()`는 원래 PC의 표준 출력(stdout)으로 문자를 내보내는데, MCU에는 stdout이 없다. STM32 + GCC 환경에서는 `printf()`가 내부적으로 `__io_putchar()`를 한 글자씩 호출하도록 설계되어 있으므로, 이 함수를 UART 전송으로 재정의(override)하면 `printf()` 출력이 UART로 나오게 된다.

```c
int __io_putchar(int ch) {
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

`#ifdef __GNUC__`는 GCC 컴파일러를 사용할 때만 이 정의를 적용하고, 다른 컴파일러(Keil 등)에서는 `fputc`를 재정의하도록 분기하는 전처리기 조건문이다.

### `--specs=nano.specs`와 float 출력

STM32CubeIDE는 기본적으로 `--specs=nano.specs`를 링커 옵션으로 사용한다. 이는 메모리가 작은 MCU를 위해 경량화된 C 표준 라이브러리(newlib-nano)를 사용하겠다는 설정이다. 이 경량 라이브러리는 용량을 줄이기 위해 **float 포맷(`%f`, `%.2f` 등) 출력 기능을 기본적으로 제거**해둔다. 따라서 링커에 `-u _printf_float` 플래그를 추가하여 float 출력 기능을 명시적으로 포함시켜야 한다. 이 옵션 없이 `%.2f`를 사용하면 빈 문자열이 출력된다.

### AO/DO 핀 차이와 내장 비교기 LM393

토양 수분 센서 모듈(SZH-EK106)에는 두 종류의 출력 핀이 있다.

**AO (Analog Output)** 핀은 수분량에 비례한 연속적인 전압을 출력한다. 건조할수록 높은 전압(저항 큼 → 전압 분배 커짐), 습할수록 낮은 전압이 나온다. MCU의 ADC로 읽어서 수분 정도를 수치로 파악할 수 있다.

**DO (Digital Output)** 핀은 모듈에 내장된 LM393 비교기(Comparator)가 AO 전압을 기준 전압과 비교하여 HIGH 또는 LOW만 출력한다. 가변저항(트리머)으로 기준 전압을 조절하면 "이 수분 이하면 LOW" 같은 단순 임계값 감지에 쓸 수 있다. 단, 2단계 출력이라 수분의 정도를 알 수 없다. 우리 프로젝트에서는 수분량을 수치로 읽어야 하므로 AO를 사용한다.
