## 센서 모듈(SZH-EK106) 관련 메모

- 모듈에 AO / DO / GND / VCC 핀 4개 있음
- **AO 핀 사용** — 연속적인 아날로그 값 (0~4095)
- DO 핀은 내장 LM393 비교기 출력 (HIGH/LOW 2단계만)
- 모듈의 가변저항(트리머)은 DO 핀 전용 — AO 사용 시 무관
- VCC는 반드시 **3.3V** 연결 (5V 연결 시 ADC 핀 손상 위험)

## main.c 코드 구조

### printf 리디렉션 (USER CODE BEGIN 0)

```c
#ifdef __GNUC__
#define PUTCHAR_PROTOTYPE int __io_putchar(int ch)
#else
#define PUTCHAR_PROTOTYPE int fputc(int ch, FILE *f)
#endif

PUTCHAR_PROTOTYPE {
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

### ADC 캘리브레이션 (USER CODE BEGIN 2)

```c
HAL_ADCEx_Calibration_Start(&hadc1);
```

### 메인 루프 (USER CODE BEGIN WHILE)

```c
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY);
uint32_t adc_value = HAL_ADC_GetValue(&hadc1);
HAL_ADC_Stop(&hadc1);

float voltage = adc_value * 3.3f / 4095.0f;
printf("Soil ADC: %lu Voltage: %.2fV\r\n", adc_value, voltage);

HAL_Delay(1000);
```

## 이번 주 배운 것들

### CubeMX USER CODE 블록 규칙

CubeMX는 `.ioc` 파일을 기반으로 `main.c`, `stm32f1xx_it.c` 등 여러 파일을 자동 생성한다. 핀 설정이나 클럭을 바꾸고 코드를 재생성하면 이 파일들이 덮어써진다. 이때 사용자가 작성한 코드가 사라지지 않도록 CubeMX는 파일 곳곳에 `/* USER CODE BEGIN xxx */` ~ `/* USER CODE END xxx */` 블록을 만들어둔다. **이 블록 안에 작성한 코드만 재생성 후에도 보존된다.** 블록 밖에 코드를 작성하면 다음 재생성 시 삭제되므로 반드시 블록 안에 써야 한다.

### ADC (Analog-to-Digital Converter)

현실 세계의 아날로그 신호(연속적인 전압)를 MCU가 처리할 수 있는 디지털 숫자로 변환하는 장치. 토양 수분 센서는 수분량에 따라 0~3.3V 사이의 전압을 출력하는데, MCU는 전압을 직접 읽을 수 없으므로 ADC가 이를 숫자로 변환한다.

**12-bit** 해상도란 변환 결과가 0~4095(2¹²−1) 사이의 정수로 나온다는 뜻이다. 0이 0V, 4095가 3.3V에 해당하므로 전압으로 환산하면 `voltage = adc_value × 3.3 / 4095`이다.

**Single-ended** 방식은 핀 하나의 전압을 GND 기준으로 측정하는 방식이다. 반대 개념인 Differential은 두 핀의 전압 차이를 측정하는 방식으로, 노이즈에 더 강하지만 핀을 2개 사용한다. 일반 센서 읽기에는 Single-ended로 충분하다.

**폴링 방식**은 `HAL_ADC_Start()`로 변환을 시작하고, `HAL_ADC_PollForConversion()`으로 완료를 기다린 뒤, `HAL_ADC_GetValue()`로 값을 읽는 순서로 동작한다. CPU가 변환이 끝날 때까지 대기하는 방식이라 단순하지만, 그동안 다른 일을 할 수 없다. 더 복잡한 방식으로는 변환 완료 시 인터럽트를 발생시키는 방법(DMA/인터럽트 방식)이 있다.

### 클럭 트리 (Clock Tree)

STM32는 여러 주변장치(UART, ADC, GPIO 등)가 각기 다른 클럭 도메인에서 동작하며, 모든 클럭의 원천은 하나의 소스에서 시작된다.

```
HSE (외부 크리스탈, 8MHz)
  └─ PLL (×9 배수)
       └─ SYSCLK (72MHz) — CPU 동작 클럭
            ├─ HCLK (72MHz, AHB 버스) — 메모리, DMA
            ├─ APB1 (/2 → 36MHz) — UART2, SPI, I2C 등
            └─ APB2 (/1 → 72MHz) — ADC1, GPIO, UART1 등
                  └─ ADC Prescaler (/6 → 12MHz) — ADC 동작 클럭
```

ADC 최대 클럭이 14MHz이므로 72MHz를 /6하여 12MHz로 맞춘 것이다. 클럭이 너무 빠르면 ADC가 오동작하고, 너무 느리면 변환 시간이 길어진다.

Nucleo-F103RB 보드는 ST-Link 칩이 8MHz 클럭을 MCU에 공급하므로 외부 크리스탈 없이 HSE를 사용할 수 있다(HSE Bypass 모드).

### `&` (주소 연산자)와 `*` (포인터)

C에서 변수는 메모리의 특정 주소에 저장된다. `&변수`는 그 변수의 메모리 주소를 반환한다. HAL 함수들이 `&hadc1`처럼 핸들을 포인터로 받는 이유는, 함수 내부에서 구조체의 내용을 직접 수정해야 하기 때문이다. 값으로 전달하면 복사본이 전달되어 원본이 바뀌지 않는다.

```c
HAL_ADC_Start(&hadc1);   // hadc1의 주소를 전달 → 함수 내부에서 hadc1 상태 변경 가능
```

`*`는 두 가지 용도로 쓰인다. 포인터 변수를 선언할 때(`int *p`)와, 포인터가 가리키는 메모리의 값을 읽거나 쓸 때(`*p = 10`) 사용한다.

### printf 리디렉션

표준 C의 `printf()`는 원래 PC의 표준 출력(stdout)으로 문자를 내보내는데, MCU에는 stdout이 없다. STM32 + GCC 환경에서는 `printf()`가 내부적으로 `__io_putchar()`를 한 글자씩 호출하도록 설계되어 있으므로, 이 함수를 UART 전송으로 재정의(override)하면 `printf()` 출력이 UART로 나오게 된다.

```c
int __io_putchar(int ch) {
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

`#ifdef __GNUC__`는 GCC 컴파일러를 사용할 때만 이 정의를 적용하고, 다른 컴파일러(Keil 등)에서는 `fputc`를 재정의하도록 분기하는 전처리기 조건문이다.

### `--specs=nano.specs`와 float 출력

STM32CubeIDE는 기본적으로 `--specs=nano.specs`를 링커 옵션으로 사용한다. 이는 메모리가 작은 MCU를 위해 경량화된 C 표준 라이브러리(newlib-nano)를 사용하겠다는 설정이다. 이 경량 라이브러리는 용량을 줄이기 위해 **float 포맷(`%f`, `%.2f` 등) 출력 기능을 기본적으로 제거**해둔다. 따라서 링커에 `-u _printf_float` 플래그를 추가하여 float 출력 기능을 명시적으로 포함시켜야 한다. 이 옵션 없이 `%.2f`를 사용하면 빈 문자열이 출력된다.

### AO/DO 핀 차이와 내장 비교기 LM393

토양 수분 센서 모듈(SZH-EK106)에는 두 종류의 출력 핀이 있다.

**AO (Analog Output)** 핀은 수분량에 비례한 연속적인 전압을 출력한다. 건조할수록 높은 전압(저항 큼 → 전압 분배 커짐), 습할수록 낮은 전압이 나온다. MCU의 ADC로 읽어서 수분 정도를 수치로 파악할 수 있다.

**DO (Digital Output)** 핀은 모듈에 내장된 LM393 비교기(Comparator)가 AO 전압을 기준 전압과 비교하여 HIGH 또는 LOW만 출력한다. 가변저항(트리머)으로 기준 전압을 조절하면 "이 수분 이하면 LOW" 같은 단순 임계값 감지에 쓸 수 있다. 단, 2단계 출력이라 수분의 정도를 알 수 없다. 우리 프로젝트에서는 수분량을 수치로 읽어야 하므로 AO를 사용한다.
