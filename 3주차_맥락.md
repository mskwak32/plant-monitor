## 핀 배치 (Nucleo-F103RB 기준)

| 기능 | STM32 핀 | 보드 커넥터 |
|------|----------|------------|
| 토양습도센서 AOUT | PA0 | CN8 A0 |
| UART TX (PC→보드) | PA2 | CN9 D1 |
| UART RX (보드→PC) | PA3 | CN9 D0 |

UART는 ST-Link 내장 USB-UART와 자동 연결 — 별도 선 불필요

---

## 센서 모듈(SZH-EK106) 관련 메모

- 모듈에 AO / DO / GND / VCC 핀 4개 있음
- **AO 핀 사용** — 연속적인 아날로그 값 (0~4095)
- DO 핀은 내장 LM393 비교기 출력 (HIGH/LOW 2단계만)
- 모듈의 가변저항(트리머)은 DO 핀 전용 — AO 사용 시 무관
- VCC는 반드시 **3.3V** 연결 (5V 연결 시 ADC 핀 손상 위험)

## 센서 ADC 측정값 (실측)

| 상태 | ADC 범위 |
|------|----------|
| 공기 중 (건조) | 4000 이상 |
| 물티슈 감쌈 (촉촉) | 1700 ~ 2500 |

> 실제 화분 흙 기준값은 추후 측정 필요

### 임계값 후보 (잠정)

| 구간 | 판단 |
|------|------|
| 3000 이상 | 건조 → 물 줘야 함 |
| 2500 이하 | 충분히 습함 → 펌프 OFF |
| 2500 ~ 3000 | 경계 구간 (애매) |

> 7주차 임계값 기반 자동 제어 구현 시 확정 예정

## main.c 최종 코드 구조

### printf 리디렉션 (USER CODE BEGIN 0)

```c
#ifdef __GNUC__
#define PUTCHAR_PROTOTYPE int __io_putchar(int ch)
#else
#define PUTCHAR_PROTOTYPE int fputc(int ch, FILE *f)
#endif

PUTCHAR_PROTOTYPE
{
    HAL_UART_Transmit(&huart2, (uint8_t*)&ch, 1, HAL_MAX_DELAY);
    return ch;
}
```

- `--specs=nano.specs` 환경에서는 `fputc` 대신 `__io_putchar`를 재정의해야 printf가 동작함
- `#ifdef __GNUC__`로 컴파일러 종류에 따라 자동 선택

### ADC 캘리브레이션 (USER CODE BEGIN 2)

```c
HAL_ADCEx_Calibration_Start(&hadc1);
```

- 부팅 시 ADC 내부 오차 자동 보정
- MX_ADC1_Init() 호출 후, while(1) 진입 전에 1회 실행

### 메인 루프 (USER CODE BEGIN WHILE)

```c
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY);
uint32_t adc_value = HAL_ADC_GetValue(&hadc1);
HAL_ADC_Stop(&hadc1);

float voltage = adc_value * 3.3f / 4095.0f;
printf("Soil ADC: %lu Voltage: %.2fV\r\n", adc_value, voltage);

HAL_Delay(1000);
```

## 링커 설정

float 출력(`%.2f`) 사용을 위해 아래 플래그 추가됨:
- **Properties → C/C++ Build → Settings → MCU GCC Linker → Miscellaneous → Other flags**
- 추가값: `-u _printf_float`

## 이번 주 배운 것들

- CubeMX USER CODE 블록 규칙 (블록 밖 코드는 재생성 시 삭제됨)
- ADC Single-ended / 12-bit / 폴링 방식
- 클럭 트리: HSE → PLL → SYSCLK → HCLK → APB1/APB2 → ADC
- `&` (주소 연산자) / `*` (포인터 선언 & 역참조)
- printf 리디렉션 (`__io_putchar`)
- `--specs=nano.specs` 환경에서 float 출력 방법 (`-u _printf_float`)
- 센서 모듈 AO/DO 핀 차이 및 내장 비교기(LM393) 동작 원리
- 가변저항(트리머)은 DO 전용이며 AO와 무관함
